## Technical Notes

### File Formats
- **Markdown (.md):** Individual railroad profiles, README, documentation
- **CSV (.csv):** Structured data for database import and analysis
- **HTML (.html):** Interactive report with embedded JavaScript for sorting

### Data Standards
- **Route Miles:** Reported as stated by railroad (may include trackage rights)
- **Class I Connections:** Listed by junction city/location where available
- **Commodities:** Listed in priority order when known
- **Dates:** All data current as of February 2026
- **Reporting Marks:** AAR standard reporting marks used throughout

### Quality Control
- Multiple sources cross-referenced for each data point
- Official G&W sources prioritized
- Data gaps explicitly noted rather than guessed
- Consistent formatting across all profiles
- Links to sources provided for verification

## Lessons Learned

### What Worked Well
- Official G&W website provided good high-level information
- Wikipedia list of G&W railroads was accurate starting point
- Industry resources (American-Rails, HawkinsRails) provided operational detail
- Structured markdown format made profiles easy to create and maintain
- CSV format enables easy data analysis and visualization

### Challenges Encountered
- **Scale:** 113 railroads is significantly larger than Watco's 40
- **Limited Public Data:** Many smaller railroads have minimal online presence
- **Fragmented Information:** Data scattered across multiple sources
- **Proprietary Data:** Customer names and traffic volumes often not public
- **Organizational Complexity:** OHCR system (10 subsidiaries) requires separate treatment

### Solutions Implemented
- **Prioritization:** Focused on largest and most strategically important railroads first
- **Framework First:** Created structure that can be filled in over time
- **Documentation:** Explicitly noted data gaps rather than guessing
- **Flexibility:** CSV format allows easy updates as new information becomes available
