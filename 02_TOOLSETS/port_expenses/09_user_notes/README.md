# 09 — User Notes

**Sub-module:** `02_TOOLSETS/port_expenses/09_user_notes/`
**Status:** Active — populate as you go | **Created:** 2026-03-02

---

## Purpose

William's working knowledge repository for port expenses — empirical rate observations, port-specific intelligence, operator notes, and reference benchmarks built from 30+ years of maritime consulting.

This is NOT a calculator. It is a structured knowledge base that calibrates and validates the quantitative sub-modules (01–08, 10–11).

## Contents

| File | Description |
|---|---|
| `port_notes.md` | Port-by-port observations — quirks, local practices, key contacts |
| `rate_benchmarks.md` | Empirical rate ranges from William's experience (not sourced from tariffs) |
| `operator_notes.md` | Tug operators, agents, stevedores — William's assessment and contacts |
| `charter_party_notes.md` | How different charter party types (voyage, time, bareboat) affect cost allocation |
| `commodity_handling_notes.md` | Commodity-specific handling quirks that affect costs |
| `reference_pdas.md` | Anonymized real PDA examples for calibration |

## How to Use

When building any calculator sub-module, check `09_user_notes/` first:
- Do the calculated rates align with William's empirical benchmarks?
- Are there port-specific rules that override the general model?
- Are there known operators/agents to pre-populate into directories?

## Guidelines

- All entries attributed to William unless noted otherwise
- Flag as `[EMPIRICAL]`, `[TARIFF]`, or `[BENCHMARK]` for traceability
- Dates matter — rate notes should include approximate year
- Client-specific data stays in `03_COMMODITY_MODULES/{commodity}/reports/{client}/` — not here
