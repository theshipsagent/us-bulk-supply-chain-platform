## Orientation for Toolset Development

### What "Master File" Means
The `phase_07_output.csv` is the canonical dataset. Every query, report, or analysis reads from this one file. The pipeline is the factory that produces it; the toolset is what consumes it.

### Toolset Candidates (based on queries already done)

1. **Port Query Tool** — "What comes into [port/region/state]?" with commodity filter
   - Already demonstrated: cement by California ports, cement by Great Lakes ports
   - Generalize to: any commodity, any geography, with tonnage rollup

2. **Commodity Summary Tool** — "Show me all [commodity] trade flows"
   - Origin countries -> US ports, with tonnage and record counts
   - Top shippers/consignees per commodity

3. **Party Lookup Tool** — "Who ships [commodity] to [port]?"
   - Shipper/consignee analysis filtered by commodity + port
   - Feeds into party harmonization refinement

4. **Pipeline Re-run Tool** — "Re-run from phase N"
   - Delete phase_N_output.csv through phase_07_output.csv
   - Run phases N through 7 sequentially
   - Useful when dictionary rules change

5. **Dictionary Editor Tool** — "Add rule for [keyword] -> [classification]"
   - Append to cargo_classification.csv
   - Re-run from phase 04

### Architecture Suggestion for Toolset
```python
# master.py — single entry point

# Each tool reads phase_07_output.csv (or a cached/indexed version)
```
